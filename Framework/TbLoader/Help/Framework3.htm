• for each document or procedure within the application, select the reports viewable other than the predefined ones;
  • define customized format and font styles to add to the predefined ones;
  • copy into the application structure reports, images and texts located both outside or within it;
  • create and save within the application structure text documents with .txt extension and read them later;
  • view enums data type properties.

It is possible to allow access the above mentioned functionalities to all the company users or to limit some of them to a restricted group of users.

Other framework tools area accessible from the several application windows. Among them are, for example, comprised:

  • the browser, that permits to dynamically navigate within all the application documents depending on criteria defined by customized parametric queries;
  • the query manager, that permits to define which selection and sorting to use to navigate among the application documents;
  • the radar, that visualizes in a tabular format data from inside any documents archive (such as data related to stored sells invoices) allowing also to customize both the graphical design and the viewed content;
  • TaskBuilder Explorer, that permits to visualize and manage (open, save, delete and rename) the application objects.
